▾

test-driven-developmentTDD 開發流程
▾
功能嚴格的 RED-GREEN-REFACTOR TDD 循環，先寫測試再寫實作
前置需求專案已有測試框架（Jest, pytest, etc.）
標籤免費
流程範例RED — 先寫失敗的測試： test("滿 1000 打 9 折", () => { const cart = new Cart([{ price: 600 }, { price: 500 }]); expect(cart.total()).toBe(990); }); → 執行測試 → FAIL GREEN — 寫最少的程式碼讓測試通過： total() { const sum = this.items.reduce((s, i) => s + i.price, 0); return sum >= 1000 ? sum * 0.9 : sum; } → 執行測試 → PASS REFACTOR — 重構但保持測試通過 → 執行測試 → PASS
systematic-debugging四階段系統化除錯
▾
功能收集證據 → 形成假設 → 驗證 → 修復
前置需求無
標籤免費
流程範例Phase 1：收集證據 - 檢查瀏覽器 console 錯誤 - 檢查 Network tab 的 API 回應 Phase 2：形成假設 假設 A：Token 過期時間設定太短 假設 B：Cookie SameSite 屬性導致跨域失效 Phase 3：逐一驗證 ✗ 假設 A：Token TTL 為 1 小時，正常 ✓ 假設 B：SameSite=Strict 導致 cookie 未攜帶 Phase 4：修復 → 將 SameSite 改為 Lax，加入 Domain 設定
